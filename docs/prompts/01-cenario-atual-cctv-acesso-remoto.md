# Prompt

Criar uma figura diagramática académica, horizontal, limpa e rigorosa, para dissertação de mestrado, em estilo vetorial técnico, com paleta de cor muito reduzida e adequada a contexto científico: tons de cinzento, preto, branco e um único tom de destaque muito discreto, como azul-acinzentado. A imagem deve ser print-friendly, legível em A4, com muito espaço em branco, alinhamento rigoroso, composição equilibrada e leitura clara da esquerda para a direita. Não usar gradientes, brilho, efeitos 3D, sombras fortes nem qualquer estética promocional.

Objetivo da figura: representar o cenário atual de acesso remoto a sistemas CCTV, mostrando dois modos correntes de acesso remoto e as respetivas consequências comuns. A figura não deve mostrar qualquer solução proposta; deve enquadrar apenas o problema atual.

Estrutura visual em três zonas, da esquerda para a direita:

1. Zona esquerda — dois cenários de acesso remoto

Mostrar dois subcenários separados, paralelos e claramente identificáveis:

A) Cenário 1 — câmaras com cartão SIM / 4G e acesso P2P
- Mostrar um pequeno grupo de câmaras IP.
- Associar visualmente a essas câmaras um símbolo claro de cartão SIM.
- Indicar conectividade móvel / 4G.
- O fluxo de acesso remoto deve sair das câmaras, passar pela Internet e seguir para um servidor intermédio do fabricante.
- Representar esse elemento como “Servidor intermédio do fabricante”, “Cloud do fabricante” ou “Relay P2P”.
- Depois do servidor intermédio, o fluxo segue para o utilizador remoto / operador.
- A representação deve sugerir dependência de mediação externa, sem dramatização excessiva.

B) Cenário 2 — acesso remoto via router e firewall com regra aberta / port forwarding
- Mostrar outro grupo de câmaras IP ligado a um router.
- Junto ao router, incluir um símbolo de firewall.
- Indicar visualmente, de forma técnica e não sensacionalista, uma regra aberta / port forwarding / serviço exposto para acesso remoto.
- O fluxo deve seguir das câmaras para o router/firewall, depois pela Internet, e daí diretamente para o utilizador remoto / operador.
- Este cenário deve sugerir acesso remoto direto e exposição de superfície de ataque, sem afirmar de forma absoluta que qualquer regra de firewall é insegura.

2. Zona central — Internet e mediação técnica

- Representar a Internet como elemento central entre origem e acesso remoto.
- No cenário SIM/4G-P2P, destacar a passagem obrigatória por servidor intermédio do fabricante.
- No cenário router/firewall, destacar o acesso remoto direto através da Internet.
- Usar setas simples, claras e coerentes, com diferenciação visual suficiente entre “via intermediário” e “acesso direto”.

3. Zona direita — consequências comuns

Fazer convergir visualmente ambos os cenários para uma área comum de consequências, apresentada como blocos, etiquetas ou lista técnica com ícones discretos. Incluir:
- “Exposição / aumento da superfície de ataque”
- “Controlo e posse de dados potencialmente incompletos”
- “Dependência de terceiros”
- “Múltiplas plataformas de acesso”
- “Ausência de gestão unificada”
- “Fragmentação operacional e de interoperabilidade”

Usar apenas rótulos curtos em português. Exemplos de labels aceitáveis: “Câmara IP”, “SIM / 4G”, “Router”, “Firewall”, “Regra aberta”, “Internet”, “Relay P2P”, “Cloud do fabricante”, “Operador”, “Acesso direto”, “Dependência de terceiros”, “Sem gestão unificada”. O estilo deve ser técnico, neutro e rigoroso, semelhante a uma figura científica contemporânea, com linhas finas e consistentes, caixas retangulares simples, ícones lineares discretos e mínimo ruído visual. Não mostrar a solução FUSE, apenas o problema atual.
